Review and refine the task checklist for a plan.

## Instructions

1. List the plan docs in `plans/` and ask which one to work on (or accept a name as argument: $ARGUMENTS).
2. Read the plan doc.
3. Assess the current task list:
   - Are tasks small enough to be individually completable? Break down anything that's more than ~30 min of work.
   - Are they in a logical build order?
   - Are there missing tasks (e.g. project setup, initial commit, deploy)?
   - Mark any that are already done based on the project state.
4. Propose the updated task list and get confirmation before saving.

Keep tasks concrete and actionable — "Set up Next.js project with Tailwind" not "Initialize frontend".
